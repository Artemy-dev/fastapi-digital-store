## Описание 

Backend API для цифрового магазина.
Реализован на FastAPI с использованием PostgreSQL и Redis.
Поддерживает JWT-аутентификацию, OAuth2 (Google, Yandex), кеширование данных и контейнеризацию через Docker.

**Аутентификация:**
- JWT токены в HttpOnly cookies
- OAuth2 авторизация через Google/Yandex
- Пароли хранятся в виде bcrypt-хэшей

**Защита регистрации:**
- Проверка доменов email через blacklist
- Данные кешируются в Redis с TTL для снижения нагрузки на БД
- Регистрация пользователей ускорена за счёт кеширования с 1068 ms до 114 ms (ускорение на 90%)

**Асинхронные задачи:**
- Интеграция Celery для фоновых задач.
- Отправка email-уведомлений и нотификаций.

**Инфраструктура:**
- Миграции через Alembic
- Postgres и Redis разворачиваются через docker-compose
- Health-check эндпоинты для мониторинга

## Бизнес-логика и процессные правила

### Пользователи
- Регистрация пользователей проверяет домен email через кеш Redis для блокировки временных почтовых сервисов.  
- Пароли хранятся в виде bcrypt-хэшей.  
- Аутентификация поддерживает JWT (HttpOnly cookies) и OAuth2 (Google/Yandex).  
- Пользователи с существующими заказами не могут удалить аккаунт.  

### Продукты
- CRUD для цифровых товаров: создание, чтение, обновление, удаление.  
- Каждое изменение товара обновляет базу данных и возвращает актуальное состояние объекта.  
- Файлы товаров хранятся как ссылки (`file_url`) для цифровой доставки.  

### Корзина
- Один пользователь может добавлять несколько товаров в корзину.  
- При добавлении одного и того же товара увеличивается `quantity`, вместо создания дубликата.  
- При удалении товара из корзины уменьшается `quantity`, а при достижении 0 элемент полностью удаляется.  

### Заказы
- Создание заказа происходит на основе текущей корзины пользователя.  
- Сумма заказа вычисляется автоматически как сумма всех товаров с учётом их количества и цены.  
- После создания заказа корзина пользователя очищается.  
- Заказы имеют статус `pending` и привязаны к пользователю, который их создал.  

### Асинхронность и кеширование
- Redis используется для кеширования заблокированных доменов и ускорения проверки регистрации.  
- Celery интегрирован для фоновых задач, включая отправку уведомлений и email.  
- Асинхронные операции оптимизируют отклик API и снижают нагрузку на базу данных.

## Запуск проекта

Установка зависимостей:
```bash
pip install -r requirements.txt
```

Запуск инфраструктуры:
```bash
docker-compose up -d
```

Применение миграций:
```bash
alembic upgrade head
```

Запуск приложения:
```bash
uvicorn app.main:app --reload
```

## Модель данных

- **User:** пользователь, поддержка ролей (is_admin)
- **Product:** товар с цифровой доставкой (file_url)
- **Cart:** позиции корзины
- **Order:** заказ с суммой и статусом
- **BlockedEmailDomain:** запрещённые домены

### Связи между сущностями

**User 1 ─── * Cart * ─── 1 Product**
<br>Один `User` может иметь много записей в `Cart`, и каждая запись в `Cart` относится к одному `Product`

**User 1 ─── * Orders**
<br>Один `User` может иметь много `Orders`, но каждый `Order` принадлежит ровно одному `User`
